# 基于本地大语言模型的法律法规智能问答系统
## 项目报告

---

## 摘要

本报告介绍一个完全本地化运行的法律法规智能问答系统。系统以国家法律法规数据库、北大法宝等公开法律数据源为基础，通过自研爬虫构建本地法规知识库，结合 FAISS 向量检索与本地大语言模型（Ollama + Qwen2.5），并引入 LangChain 多 Agent 调度框架，实现支持多轮交互、任务调用、带引用来源的法律问答。系统所有数据、向量索引与推理均在本地完成，无需云端依赖，兼顾可解释性与可扩展性。

**关键词**：本地大模型；RAG；FAISS；多 Agent；法律法规问答

---

## 第一章 项目背景与意义

随着大模型技术的快速发展，法律领域对智能问答与信息抽取提出了更高要求。面对海量、分散、更新频繁的法律法规文档，传统关键词检索难以满足"精准定位法条并生成可解释答案"的需求。

本项目的研究与实践意义在于：
- 验证本地大语言模型在法律垂直领域落地的可行性；
- 构建"自爬知识库 + 向量检索 + RAG + 多 Agent"的完整技术链路；
- 通过引用来源增强回答的可解释性与可信度；
- 全程离线运行，保障数据隐私与合规。

---

## 第二章 技术路线与工具选型

| 模块 | 技术选型 | 说明 |
|------|----------|------|
| 本地 LLM | Ollama + Qwen2.5（3b/7b/14b） | 本地推理，模型可热切换 |
| 文档爬取 | Python（requests + 官方 API） | 国家法律法规数据库、北大法宝 |
| 文本分段 | 自研 `LegalChunker` | 长拆短扩 |
| 向量化 | BAAI/bge-large-zh-v1.5（1024 维） | CPU 推理 |
| 检索 | FAISS（IndexFlatIP） | 余弦相似度 |
| Agent 框架 | LangChain 1.x（LangGraph） | 多 Agent 协作 |
| 系统接口 | FastAPI | RESTful + Swagger |
| 前端 | Streamlit | 可视化问答 |

技术路线遵循"数据层 → 检索层 → 生成层 → 调度层"四层架构，前端经 API 与后端交互。

---

## 第三章 系统总体设计

系统由六个核心模块构成：

1. **数据爬取模块（crawler）**：从公开法律数据源下载并清洗文本；
2. **文本分段与向量化模块（rag）**：法条分块、Embedding 编码；
3. **向量索引与检索模块（rag）**：FAISS 索引构建与语义检索；
4. **多 Agent 调度模块（agents）**：三种协作模式编排；
5. **API 服务模块（api）**：FastAPI 暴露问答与检索接口；
6. **可视化前端模块（frontend）**：Streamlit 交互界面。

数据流：原始文本 → 清洗分段 → 向量化 → FAISS 索引 → 检索 → 上下文 → 本地 LLM 生成 → 带引用回答。

## 第四章 系统实现

### 4.1 数据爬取与知识库构建
实现国家法律法规数据库爬虫（搜索 → 详情 → 下载 DOCX → 解析）与北大法宝爬虫，清洗后仅保留宪法与法律，过滤地方法规、司法解释与案例。最终形成 84 部核心法律、8798 条原始分段、8835 条向量。

### 4.2 向量化与检索
采用 `bge-large-zh-v1.5` 生成 1024 维归一化向量，FAISS `IndexFlatIP` 做余弦检索；检索结果经"同法条折叠去重"，避免重复法条占用上下文。

### 4.3 RAG 问答核心
用户问题向量化后检索 top_k 法条，拼装为"法律名 + 条款编号 + 正文"的结构化上下文，经统一管理的 Prompt 模板驱动本地 LLM 生成带引用来源的答案。

### 4.4 多 Agent 调度
基于 LangChain 1.x 实现三种协作模式：
- **串行（sequential）**：检索 Agent → 问答 Agent；
- **并行（parallel）**：LLM 拆子问题 → 并发检索 → 问答 Agent 汇总；
- **合作（hierarchical）**：调度 Agent 自主决策调用检索 / 问答 / 总结子 Agent。

支持以 `session_id` 维持多轮对话上下文。

### 4.5 API 与前端
FastAPI 提供 `/chat`、`/search`、`/health`、`/clear_session` 端点，自动生成 Swagger 文档；Streamlit 前端支持模型与模式切换、对话式交互。

## 第五章 系统测试与评估

- 构造结构化标注测试集，每条含 `question` / `expected_answer` / `category` 等字段，覆盖多个法律类别，满足"每组至少 100 条"的要求；
- 提供批量评估脚本 `evaluation/run_eval.py`，自动填充 `retrieved_text` 与 `model_output`；
- 提供自动评分脚本 `evaluation/auto_eval.py`，从准确性、相关性等维度给出评价。

测试集格式示例：
```json
[
  {
    "question": "试用期最多可以约定多久？",
    "expected_answer": "依据《劳动合同法》第十九条，劳动合同试用期不得超过六个月。",
    "category": "劳动法"
  }
]
```

## 第六章 系统演示与运行

系统采用"后端 API + 前端"架构，演示时同时启动：
- 后端：`python scripts/run_api.py`（默认 `http://localhost:8000`）；
- 前端：`streamlit run frontend/streamlit_app.py`（默认 `http://localhost:8501`）。

用户在前端选择模型与 Agent 模式，输入法律问题即可获得带引用来源的答案；亦可通过 Swagger UI 直接调试各 API 端点。

---

## 第七章 总结与展望

本项目实现了"自爬文档 → 向量索引 → RAG → 多 Agent 调度 → API → 可视化前端"的完整链路，全程本地化运行，回答可解释、架构可扩展。

后续优化方向：
- 检索策略增强（混合检索 / 重排序）；
- 更大规模语料与更细粒度评测；
- 前端交互与多模态展示优化；
- 引入更专业的法律评测基准。
