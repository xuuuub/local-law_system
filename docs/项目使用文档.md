# 法律法规智能问答系统 — 项目使用文档

本文档面向使用与部署人员，说明如何安装、配置、启动本系统，以及如何使用 API 与前端界面。

---

## 1. 环境要求

- 操作系统：Windows / Linux / macOS
- Python：3.13（建议虚拟环境）
- Ollama：已安装并运行桌面版（用于本地模型推理）
- 可选：NVIDIA GPU（加速推理，非必须）

---

## 2. 安装步骤

### 2.1 安装 Python 依赖
```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 2.2 安装并拉取本地模型
下载安装 [Ollama](https://ollama.com) 后，在终端执行：
```bash
ollama pull qwen2.5:3b      # 轻量，速度快
ollama pull qwen2.5:7b      # 推荐，效果均衡
ollama pull qwen2.5:14b     # 精度更高，资源占用大
```

---

## 3. 数据准备（首次使用）

系统已内置构建好的向量索引（`data/vectors/npc.faiss`）。如需更新或重建知识库：

### 3.1 爬取法律文本
```bash
python scripts/run_crawler.py --source npc --type law --keywords 民法典 --max 20
```

### 3.2 构建向量索引
```bash
python scripts/build_index.py
```
该命令会读取 `data/processed/npc_segments.json`，分块、编码并生成 FAISS 索引。

---

## 4. 启动系统

建议开两个终端：

### 4.1 启动后端 API
```bash
python scripts/run_api.py
```
默认监听 `http://localhost:8000`，启动时预加载索引。

### 4.2 启动前端界面
```bash
streamlit run frontend/streamlit_app.py
```
浏览器打开 `http://localhost:8501`。

---

## 5. API 接口说明

| 端点 | 方法 | 说明 |
|------|------|------|
| `/chat` | POST | 多 Agent 问答 |
| `/search` | POST | 纯向量检索（不调用 LLM） |
| `/health` | GET | 健康检查（索引大小 / 维度） |
| `/clear_session` | POST | 清空指定会话历史 |

Swagger 文档：`http://localhost:8000/docs`

### 5.1 `/chat` 请求示例
```json
{
  "question": "试用期最长可以约定多久？",
  "mode": "sequential",
  "model": "qwen2.5:3b",
  "session_id": "default"
}
```
- `mode`：`sequential`（串行）/ `parallel`（并行）/ `hierarchical`（合作）
- `session_id`：用于隔离多用户 / 多轮对话历史

### 5.2 `/search` 请求示例
```json
{
  "query": "劳动合同解除",
  "top_k": 5
}
```

---

## 6. Agent 模式说明

- **串行（sequential）**：检索 Agent 先检索法条，问答 Agent 再生成答案，流程可控，适合单点问题。
- **并行（parallel）**：将问题拆为多个子问题并发检索后汇总，适合综合性问题。
- **合作（hierarchical）**：调度 Agent 自主决策调用检索 / 问答 / 总结子 Agent，体现多 Agent 协作。

---

## 7. 前端使用

1. 在左侧栏选择 **模型**（3b / 7b / 14b）与 **Agent 模式**；
2. 点击"系统状态"确认后端在线与索引规模；
3. 在输入框输入法律问题，或点击示例问题；
4. 点击"发送"获取带引用来源的答案；
5. 点击"清空对话"清除当前会话上下文。

---

## 8. 常见问题

| 问题 | 处理建议 |
|------|----------|
| 前端提示"无法连接 API" | 确认已启动 `python scripts/run_api.py` 且端口 8000 未被占用 |
| 回答缓慢 | 改用 `qwen2.5:3b`，或确认 Ollama 已运行 |
| 检索结果为空 | 检查 `data/vectors/npc.faiss` 是否存在，必要时重建索引 |
| 爬虫被限流 | 增大请求间隔，或降低 `--max` 数量 |
