### 3.4 RAG 问答核心

RAG（检索增强生成）是本系统回答"有据可依"的核心，由检索与生成两段式链路组成。

**检索侧**
- 用户问题经 `bge-large-zh-v1.5` 编码为 1024 维向量，在 FAISS（`IndexFlatIP`）中做余弦相似度检索，取 `top_k` 条最相关法条分块；
- `rag/retriever.py` 在检索结果上做**同法条折叠去重**：对来自同一法律同一条款的多条分块合并展示，避免重复法条浪费上下文窗口；
- `rag/prompts.py` 的 `build_context_text()` 将检索到的分块拼装为带"法律名 + 条款编号 + 正文"的结构化上下文。

**生成侧**
- 采用 Prompt 模板驱动生成，模板统一在 `rag/prompts.py` 管理，主要三类：
  - `QA_TASK_TEMPLATE`：面向问答，要求模型仅依据给定法条作答并标注引用来源；
  - `SUMMARY_TASK_TEMPLATE`：面向法条结构化总结（列要点 / 概括）；
  - `RETRIEVAL_TASK_TEMPLATE`：面向检索 Agent 的工具调用指令；
- 生成链路：结构化上下文 + 用户问题 → Prompt → 本地 Ollama（Qwen2.5）→ 带引用来源的答案；
- 该 Prompt 模板被 RAG 与 Agent 两层共用，保证检索与生成语义一致。

**RAG 主链路**
```
用户问题 → 向量编码 → FAISS 检索(top_k) → 同法条折叠去重
         → 拼接结构化上下文 → Prompt 模板 → Ollama 生成 → 答案 + 引用来源
```

### 3.5 多 Agent 调度结构

基于 LangChain 1.x（LangGraph Agent）实现多 Agent 协作调度，核心代码在 `agents/orchestrator.py`，对外统一由 `AgentOrchestrator` 类编排。

**三类角色 Agent**
| 角色 | 实现方式 | 职责 |
|------|----------|------|
| 检索 Agent | LangGraph ReAct + `law_search` 工具 | 自主决策检索，返回相关法条 |
| 问答 Agent | LCEL chain（`QA_TASK_TEMPLATE \| llm`） | 基于法条生成带引用的答案 |
| 总结 Agent | LCEL chain（`SUMMARY_TASK_TEMPLATE \| llm`） | 结构化总结法条要点 |

**三种协作模式**
- **串行（sequential）**：检索 Agent 先调 `law_search` 拿法条，问答 Agent 再基于法条生成答案；流程清晰、可控，适合单点法律问题。
- **并行（parallel）**：先用 LLM 把问题拆为 2~3 个子问题，`ThreadPoolExecutor` **真正并发**多路检索，问答 Agent 汇总多路法条作答；拆分失败自动降级为串行并在答案末尾标注。
- **合作（hierarchical）**：调度 Agent（ReAct）把检索 / 问答 / 总结包装为三个子工具，由 LLM **自主决策**调用哪个子 Agent，体现 Agent 间的动态协作。

**多轮对话与上下文**
- `chat()` 方法以 `session_id` 隔离多用户对话历史，保留最近 3~5 轮（`history_text`）拼入上下文，使后续追问能延续前文；
- `clear_session()` 支持显式清空历史，配合前端"清空对话"按钮。

**调度结构示意**
```
           用户问题 + session 历史
                    │
   ┌────────────────┼────────────────┐
   ▼                ▼                ▼
sequential       parallel        hierarchical
(检索→问答)    (拆问→并发检索→汇总)  (调度Agent→子Agent)
   │                │                │
   └────────────────┴────────────────┘
                    ▼
            带引用来源的最终回答
```

## 6. 遇到的问题与解决思路

| 问题 | 解决思路 |
|------|----------|
| 爬虫接口限流 | 请求间隔 ≥0.5s，pageSize 调大减少请求次数 |
| 长法条上下文溢出 | 分块时按项 / 句拆分并保留法条前缀 |
| Ollama 推理超时 | 缩短 timeout（180s→60s）并启用自动重试 |
| 短法条语义不足 | 分块时扩写上下文增强检索召回 |
